# Kiro 卡密系统 - 使用说明

## 🚀 快速开始

### 方案1: 本地使用(暂时的)

1. 双击 `启动服务器.bat` (如果有的话)
2. 或者在命令行运行:
   ```bash
   npm install
   npm start
   ```
3. 访问:
   - 用户页面: http://localhost:3000
   - 管理页面: http://localhost:3000/admin

**缺点**: 关机后数据会保留,但服务停止

### 方案2: 部署到Render(推荐)✨

#### 一键部署

1. 双击 `一键部署到GitHub.bat`
2. 输入GitHub仓库地址
3. 等待上传完成
4. 按照提示在Render.com创建服务

#### 手动部署

1. **创建GitHub仓库**
   - 访问 https://github.com/new
   - 创建新仓库(例如: kiro-card-system)

2. **上传代码**
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/你的用户名/仓库名.git
   git push -u origin main
   ```

3. **在Render部署**
   - 访问 https://render.com
   - 注册/登录
   - 点击 "New +" → "Web Service"
   - 选择 "Connect a repository"
   - 选择刚才创建的仓库
   - 配置:
     * **Name**: kiro-card-system
     * **Environment**: Node
     * **Build Command**: `npm install`
     * **Start Command**: `npm start`
     * **Instance Type**: Free
   - 点击 "Create Web Service"

4. **等待部署**
   - 首次部署需要3-5分钟
   - 部署成功后会显示公网地址
   - 例如: https://kiro-card-system.onrender.com

## 📱 访问系统

部署成功后:

### 用户验证页面
```
https://你的域名.onrender.com/
```
- 用户在这里输入卡密验证
- 有验证码保护
- 验证成功后显示凭证信息

### 管理页面
```
https://你的域名.onrender.com/admin
```
- 实时查看所有卡密
- 查看使用状态
- 统计信息
- 数据自动刷新(每5秒)

## 🔗 扩展集成

### 配置FlowPilot扩展

修改扩展中的API地址:

```javascript
// 在 publisher-kiro-rs.js 或相关文件中
const VERIFICATION_API = 'https://你的域名.onrender.com/api';
```

扩展会自动:
1. 生成Kiro账号
2. 生成卡密
3. 上传到验证系统
4. 卡密立即可用

### 测试连接

```javascript
// 测试API是否可访问
fetch('https://你的域名.onrender.com/api/stats')
  .then(r => r.json())
  .then(data => console.log('连接成功:', data));
```

## 💾 数据管理

### 数据存储位置

- **本地**: `cardkeys.db` 文件
- **Render**: 自动持久化存储

### 数据备份

```bash
# 导出所有卡密
curl https://你的域名.onrender.com/api/cardkeys/all > backup.json
```

### 数据恢复

如果数据丢失,可以批量重新添加:

```javascript
fetch('https://你的域名.onrender.com/api/cardkeys/batch', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    cardKeys: [...]  // 从backup.json复制
  })
});
```

## 🔧 常见问题

### Q: Render服务休眠怎么办?

A: Render免费版会在15分钟无活动后休眠。首次访问需要等待30秒唤醒。

**解决方案**:
1. 升级到付费版($7/月)
2. 使用外部监控服务定期访问(如UptimeRobot)

### Q: 数据会丢失吗?

A: 不会。SQLite数据库文件会持久化保存在Render服务器上。

### Q: 如何更新代码?

```bash
# 修改代码后
git add .
git commit -m "更新说明"
git push

# Render会自动重新部署
```

### Q: 如何查看服务器日志?

1. 登录Render.com
2. 进入你的服务
3. 点击 "Logs" 标签
4. 查看实时日志输出

### Q: 端口3000被占用?

Render会自动分配端口,代码中使用:
```javascript
const PORT = process.env.PORT || 3000;
```

### Q: 如何添加密码保护?

管理页面建议添加基础认证:

```javascript
// 在 server.js 中添加
const basicAuth = require('express-basic-auth');

app.use('/admin', basicAuth({
  users: { 'admin': 'your-password' },
  challenge: true
}));
```

## 📊 监控建议

### 使用UptimeRobot保持唤醒

1. 注册 https://uptimerobot.com
2. 添加新监控
3. URL: https://你的域名.onrender.com/api/stats
4. 间隔: 5分钟
5. 保存

这样服务器会保持唤醒状态。

### 日志监控

定期检查Render日志,确保:
- 没有数据库错误
- API响应正常
- 没有异常流量
